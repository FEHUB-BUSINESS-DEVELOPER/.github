# Política de Seguridad

## Versiones soportadas

Solo se dan parches de seguridad a la rama `main` y al último release.

## Reportar una vulnerabilidad

Si encuentras una vulnerabilidad, **no la reportes públicamente**. Envía un correo a **carlos.rodriguez@feduro.net** o **martin.gomez@feduro.net**  con:

- Descripción del problema
- Pasos para reproducir
- Impacto potencial
- Versión / commit afectado

Nos comprometemos a responder en un plazo de 48 horas hábiles.

## Buenas prácticas

- Nunca commits credenciales, tokens o llaves.
- Usa variables de entorno y GitHub Secrets.
- Revisa las alertas de Dependabot y aplícalas.
